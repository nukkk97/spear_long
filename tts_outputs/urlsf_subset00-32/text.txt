An linter runs as you modify your source code to try to highlight code that could possibly be improved The linter can be turned off from the Power Tools general settings where it is listed as FSharpLint Integration When broken lint rule is discovered the first two characters of the code in question are underlined in orange mouse over of these characters will show the lint suggestion in tooltip Each rule can be turned off in the Power Tools lint settings under the Rules tab the checkboxes indicate whether rule is enabled disabled Some rules are grouped in collection under an analyser disabling the analyser will disable all of its rules to enable disable specific rules expand the analyser to view its rules Some rules and analysers have additional settings these can be viewed and updated by selecting the rule analyser Hints in the linter provide way for new rules to easily be added by modifying the configuration alone They are written in DSL that appears as subset of in the format of match expression suggest expression or suggest message Further documentation on writing hints Under the Hints tab in the Power Tools lint settings existing hints can be viewed and new hints can be add
